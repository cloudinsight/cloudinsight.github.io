Cloudinsight SDK For Node.js

Cloudinsight 根据 StatsD 进行了修改，形成了 Cloudinsight SDK，可以方便用户上传自定义指标至 Cloudinsight。

目前支持的数据类型有以下几种，具体参考 SDK 数据类型：

  • Counters
  • Gauges

使用 SDK 的前提条件为您已经安装了 Cloudinsight Agent。Agent 的采集周期为 30 s，数据发送周期为 15 s。

下载与安装

  • 使用 npm 安装

    npm install cloudinsight-sdk

数据类型

Counters

Counters 是 StatsD 和 Cloudinsight SDK 最基本的一种数据类型。当调用 increment 和 decrement 时，对参数进行自增或自减。

由于 Cloudinsight Agent 的采集周期为 30 s，最终展现至界面的值为采集周期的平均值。

// Require the module
var StatsD = require( 'cloudinsight-sdk' ).StatsD;

// Instantiate the StatsD, with host, port and global tags specified.
// The default host & port is 'localhost:8251'
var statsd = new StatsD( '127.0.0.1', 8251, [ 'machineName:awsBeijing1', 'timezone:GMT8' ] );

// Increment a counter.
statsd.increment('page.views');

increment 的步进为 1，也可以调用incrementBy调整步进

statsd.incrementBy('page.views', 10);

采样率

可以通过 sample_rate 参数来设置采样率。假设 sample_rate=0.1：

采样率为在采集周期 30 s 内，只采集 10% 的数据。也就是说，调用了 10 次 increment，只自增了一次 Metric。

在数据量较大的情况下，可以确保采样的高效。

  statsd.increment('page.views', 1, ['page.tag'], 0.1)

采样率会传递至 Agent，所以最终发送的数值，为计算采样率后的真实数值。

同样，decrement 和 increment 一样，使用相同的参数。

  # Decrease the counter
  statsd.decrement('some.other.event')

  # Decrease the counter by 5, 15% sample.
  statsd.decrementBy('some.third.event', 5, ['some.tag'], 0.15)

Gauges

Gauges 是连续型的数据类型，也就是说，当我们第一次赋值后，该 Metric 表现在图表上，会是一条于 X 轴平行的直线；直到再次为该 Metric 赋了一个不同的值。

Gauges 可以用来度量一些已经计算过的数值，如 WebSockets 的有效链接数；而监控系统目前登录的用户数，则在每次用户登录时，调用 Counters 中的 increment 比较适合。

  # Record a gauge
  statsd.gauge('foo.bar', 123)

此时，foo.bar 的值为 123。

Metric Name

命名 Metric Name 有以下一些规则：

  • 指标名称必须以字母为开头
  • 只能包含 ASCII 字母，数字，下划线和句点（其它字符将被转换为下划线）
  • 最多200个字符（从 UI 考虑不要超过100）
  • 不支持 Unicode
  • 建议避免空格
